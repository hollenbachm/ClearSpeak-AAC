# ClearSpeak Roadmap

ClearSpeak is a simple AAC (Augmentative and Alternative Communication) tool designed to help stroke survivors and others with speech impairments communicate basic needs and phrases more easily.

This roadmap outlines current priorities and longer-term goals for the project.

---

## Phase 1: Stabilize the Prototype

**Goal:** Improve reliability, clarity, and ease of use for real-world communication.

### Core usability
- [x] Add Quick Needs board
- [x] Add sentence builder
- [x] Add Easy Mode
- [x] Add caregiver customization
- [x] Add My People / My Routine
- [x] Add partner communication tips
- [x] Add voice settings
- [x] Add local storage persistence
- [x] Increase tap target sizes where needed
- [x] Improve visual separation between sections
- [ ] Review contrast and readability for accessibility
- [ ] Confirm speech output works reliably across supported browsers and devices
- [ ] Add a **Stop Speaking** button
- [ ] Test local storage persistence across reloads
- [ ] Confirm mobile usability on Android and iOS browsers

### Refinements
- [ ] Allow caregivers to reorder Quick Needs cards
- [ ] Allow caregivers to mark favorite cards
- [ ] Add large text / large touch target mode
- [ ] Improve sentence builder clarity and flow
- [ ] Continue reducing clutter and unnecessary friction

---

## Phase 2: Expand Caregiver and Personalization Tools

**Goal:** Make the app easier to customize for individual users.

### Caregiver tools
- [ ] Add editing for existing custom cards
- [ ] Add editing for My People / My Routine items
- [ ] Allow caregivers to hide unused categories
- [ ] Add caregiver lock mode to prevent accidental changes
- [ ] Add import/export for card sets and user configurations

### My People / My Routine
- [x] Add My People / My Routine section
- [x] Allow adding personalized items
- [ ] Support custom photos for familiar people and important objects
- [ ] Add more personalized quick-access phrases
- [ ] Expand personalization for:
  - [ ] People
  - [ ] Places
  - [ ] Items
  - [ ] Food and drink
  - [ ] Daily routines

---

## Phase 3: Improve Communication Speed and Clarity

**Goal:** Make common communication faster and more intuitive.

### Communication features
- [ ] Add pinned phrases for most-used needs and requests
- [ ] Add an emergency phrases section
- [ ] Add a full-screen Yes / No response mode
- [ ] Add a pain scale board
- [ ] Add body location cards for pain and symptoms
- [ ] Explore phrase suggestions after selecting sentence starters
- [ ] Reduce the number of taps required for common requests

---

## Phase 4: Expand Accessibility and Practice Tools

**Goal:** Support both communication and speech/language practice.

### Accessibility
- [ ] Add reduced clutter mode
- [ ] Add optional color-coded categories
- [ ] Improve tablet-first layout
- [ ] Continue refining spacing, text size, and touch accessibility

### Practice mode
- [x] Add basic practice mode
- [ ] Expand simple therapy-style exercises:
  - [ ] Picture naming
  - [ ] Word-to-picture matching
  - [ ] Phrase repetition
  - [ ] Sentence completion
  - [ ] Simple comprehension prompts
- [ ] Keep exercises adult-appropriate and easy to understand
- [ ] Continue aligning practice content with real-world communication needs

---

## Phase 5: Packaging and Distribution

**Goal:** Prepare ClearSpeak for broader use and easier access.

### Web app improvements
- [x] Publish live prototype
- [x] Create GitHub repository
- [ ] Convert ClearSpeak into a Progressive Web App (PWA)
- [ ] Add installable home screen support
- [ ] Improve offline readiness
- [ ] Continue testing across browsers and devices

### Android distribution
- [ ] Prepare for Android packaging once the web experience is stable
- [ ] Explore Trusted Web Activity packaging for Google Play
- [ ] Create app store assets when the project is ready for wider release

---

## Phase 6: Long-Term Growth

**Goal:** Explore larger-scale features only if the project grows and real-world demand supports them.

### Possible future features
- [ ] Cloud sync for caregiver-managed profiles
- [ ] Multiple user profiles on one device
- [ ] Phrase usage analytics
- [ ] Therapist or caregiver dashboards
- [ ] Shared profile setup across multiple devices
- [ ] Expanded multilingual support

---

## Feedback Priorities

The most valuable feedback areas for ClearSpeak are:

- [ ] AAC layout and usability feedback
- [ ] Stroke recovery communication needs
- [ ] Aphasia-friendly design guidance
- [ ] Caregiver workflow feedback
- [ ] Speech-language pathology input
- [ ] Accessibility and assistive technology best practices

---

- [x] Keep the app free to use
- [x] Support development through optional donations instead of a required subscription
- [x] Keep the project open source
- [ ] Continue refining the app based on real-world feedback
- [ ] Reach out to rehabilitation and speech-language professionals for practical guidance
